STOUT

Accounting Software from Scratch — Making inherently dense financial data navigable without sacrificing power user efficiency.

Overview

I was the sole designer building an accounting platform from the ground up. The challenge: make inherently dense financial data navigable without sacrificing efficiency for power users.

The Problem

Accounting systems are information-dense by nature. Users need quick access to financial statements, transaction histories, and reconciliation tools—but existing interfaces buried critical functions in deep menu hierarchies. Target users were experienced accountants who valued speed over hand-holding.

My Role

Solo Product Designer

End-to-end design ownership: strategy, information architecture, wireframes, UI design, interaction patterns, and documentation. Worked directly with stakeholders to define priorities and validate decisions.

Deliverables

  • Complete UI system for financial workflows
  • Transaction management interfaces
  • Financial statement generators
  • Reconciliation tools
  • Reporting dashboards with advanced filters

Design Approach

  • Flat navigation: Critical functions accessible within 1-2 clicks
  • Dense but scannable: High information density with clear visual hierarchy
  • Contextual actions: Right tools appear at the right time
